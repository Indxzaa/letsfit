# Phase 14 — Sound Effects System

## Approach

Centralized `lib/audio.ts` singleton using Web Audio API (no asset files). All 18 sounds
synthesized in code via oscillators + ADSR envelopes. Mute state persisted in localStorage,
reactive via a custom event (mirrors the `progress.ts` pattern). Hover/click sounds wired via
document-level event delegation from a one-time mount component at the layout root.

---

## New Files (3)

### `lib/audio.ts`
- `SoundName` — strict union type for all 18 sounds
- Single `AudioContext`, lazy-initialized on first `playSound()` call
- `playSound(name: SoundName)` — public API, importable anywhere
- Per-sound anti-spam: `Map<SoundName, number>` last-played timestamps + per-sound cooldowns
  - hover: 150 ms, click: 100 ms, rep: 200 ms, countdown: 300 ms, others: 500 ms
- Autoplay compliance: `ctx.resume()` called on first gesture via `SoundInit`
- Mute: `isMuted()`, `setMuted(v: boolean)`, `toggleMuted()`, `subscribeMute(handler)`
  - `localStorage` key `'letsfit:sound:muted'`
  - custom event `'letsfit:sound:changed'` for React reactivity
- 18 synthesized sounds:

### `hooks/useSound.ts`
- `useSoundSettings()` → `{ isMuted: boolean; toggleMuted: () => void }`
- Uses `subscribeMute` + `useState`/`useEffect`

### `components/SoundInit.tsx` (`'use client'`, renders `null`)
- Mounted once at layout root
- `mouseover` delegation with `lastHoveredButton` dedup → `playSound('hover')` on `button:not([disabled])`
- `click` delegation → `playSound('click')` on `button:not([disabled]), a[href]`
- First gesture (`pointerdown`/`keydown`/`touchstart`) → `ctx.resume()` to unlock AudioContext
