# Phase 10 — Shop Rework (Part 1)

## Scope confirmed
1. Hide Border/Aura shop tabs, keep data for backward compat.
2. Add Emoji shop category (7 items, exact prices given).
3. Emoji ownership = simple owned pool (no per-slot equip step) — this is a deliberate call: emojis are multiplayer reactions, not profile decorations, so there's nothing to "equip." Owning it via `unlockedItems` is enough to use it.
4. Realtime emoji reactions in multiplayer (Lobby, Freestyle, 1v1 Battle), broadcast-based, 2.5s client cooldown, auto-dismiss ~2.5s, top-right placement, no chat/persistence.
5. Neobrutalist styling reusing existing components/patterns.
6. Explicitly NOT touching crates/chests/frames.

## Files

### 1. `lib/shop.ts`
Add an `EMOJI_ITEMS` array, appended into `SHOP_ITEMS`. Border/aura arrays untouched.
```
emoji1.png → 'emoji-angry'   → Angry   → 100
emoji2.png → 'emoji-sassy'   → Sassy   → 150
emoji3.png → 'emoji-roses'   → Roses   → 250
emoji4.png → 'emoji-smile'   → Smile   → 350
emoji5.png → 'emoji-laughing'→ Laughing→ 430
emoji6.png → 'emoji-loser'   → Loser   → 670
emoji7.png → 'emoji-bulleh'  → "Why You Bulleh Meh" → 750
```
`value` = the public path (`/emoji1.png` etc). Rarity labels are my own pick (common→legendary scaling with price) since none were specified — purely a cosmetic tag color, no gameplay effect.

### 2. `lib/progress.ts`
Extend `ShopItem['type']` union: add `'emoji'`. No other changes — `purchaseItem()` is reused as-is; ownership already lives in `unlockedItems`.

### 3. `app/shop/page.tsx`
- `TABS` → only `{ id: 'emoji', label: 'Emojis' }` (border/aura tabs removed; keeping the array-driven structure so future phases can append crate/frame tabs later without restructuring).
- `CARD_COLORS.emoji` → a Neobrutalist card color (amber).
- `ItemPreview`: add an `emoji` branch rendering the actual PNG via `next/image` (`object-contain`, ~80px box) instead of an emoji-character placeholder.
- Purchase card footer: emoji items have no equip step, so once unlocked show a static "Owned ✓" state instead of the Equip/Equipped toggle used by border/aura. Locked/afford logic (purchase button, insufficient-funds lock) is reused unchanged.

### 4. `lib/multiplayer/reactions.ts` (new)
Mirrors `workoutSession.ts`'s broadcast pattern exactly, but as its own small channel so it's usable from both the Lobby and Session pages without coupling to round-based session state:
```ts
export type EmojiReactionEvent = { userId: string; emojiId: string };
function reactionChannel(roomId: string) { return `emoji_reaction:${roomId}`; }
export function subscribeEmojiReactions(roomId, onEvent): () => void { ... }
export async function broadcastEmojiReaction(roomId, event): Promise<void> { ... }
```

### 5. `components/multiplayer/EmojiReactionBubble.tsx` (new)
Pop-in/auto-dismiss visual, modeled directly on `XpPopup.tsx`'s pattern (AnimatePresence + setTimeout-based expire callback) instead of duplicating that logic. Renders the emoji PNG via `next/image`, sized `clamp(48px, 8vw, 72px)`, anchored absolute top-right with a small margin, scale+fade pop-in, auto-calls `onExpire` after ~2.5s.

### 6. `components/multiplayer/EmojiPicker.tsx` (new)
Small Neobrutalist button (matches existing `CtrlBtn` sizing/style in the session page) that opens a compact popover grid of the player's *owned* emoji items only. Tapping one:
- No-ops if still inside the 2.5s cooldown window (tracked via a local ref, not global state).
- Otherwise calls `onSend(emojiId)` and starts the cooldown.
Reused identically in both the session page and the lobby page.

### 7. `app/workout-together/session/page.tsx`
- Subscribe once in `SessionContent` via `subscribeEmojiReactions`, routing incoming events to `myReaction` / `partnerReaction` state (keyed by whether `event.userId === user?.id`) — same "lift state up, pass down as props" pattern already used for `repCount`/`repFlash`.
- `CameraPanel` gets a new optional `reaction` prop; renders `EmojiReactionBubble` inside the existing `relative` video container (line ~114) when set.
- Add the `EmojiPicker` button into the "You" panel's existing bottom-right `controls` cluster (alongside Mic/Video/Pause), wired to `broadcastEmojiReaction(roomId, { userId, emojiId })`.

### 8. `app/workout-together/lobby/page.tsx`
The lobby has no camera feed, so reactions anchor to the player's row instead of a video panel (the closest visual equivalent):
- Add the same `EmojiPicker` button in the Actions area, broadcasting on the same `emoji_reaction:{roomId}` channel.
- `LobbyContent` subscribes and keeps a `Record<userId, reaction>` map; `LobbyPlayerRow` renders an `EmojiReactionBubble` top-right of its own row when a reaction exists for that player.

## Explicitly out of scope (per spec)
Crates, chests, profile frames — not touched.
